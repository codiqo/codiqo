## DIFF CLASSIFICATION REQUIRED

This commit has [(${request.changeSummary.linesAdded})] added line(s) and [(${request.changeSummary.linesDeleted})] deleted line(s). The server has already filtered out blank / import / comment-only lines, grouped the remaining effective lines into change blocks, and will pair each block mechanically (`min(D, A)` pairs in diff order; the excess becomes pure additions / deletions). Per STEP 1a in the system prompt, you provide ONLY the semantic labels: per-block `blockKinds` (`"inPlace"` vs `"trueModify"`) and the `cosmeticAdded` / `cosmeticDeleted` exception lists.

Every effective `-` / `+` line in `fileChanges[*].diff` is annotated `-<old-file line>|B<n>|content` / `+<new-file line>|B<n>|content` — its authoritative line number and change-block id. **Copy block ids and line numbers from these tags exactly** — never count lines or derive anything from hunk headers. A `±` line without a `B<n>` tag is already excluded; unchanged context lines carry no annotation. The server rejects any cited block id or cosmetic line number that does not exist as a tag in that file's diff.
